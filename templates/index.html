Outbox Pattern Prototype

The Outbox pattern is a messaging design pattern that ensures reliable message delivery and data consistency in event-driven microservices. It works by storing messages in a dedicated "outbox" table within the source service's database, within a transaction alongside database updates. A background process then reads from this outbox and publishes the messages, ensuring that messages are sent at least once and that database updates and message publication are treated atomically.

It uses utility classes for typography and spacing to space content out within the larger container.
